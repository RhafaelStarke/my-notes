Tags Semânticas

Wireframe

<header>

Sua função principal é definir um cabeçalho ou uma seção introdutória para o seu conteúdo.

O que vai dentro da tag <header>?

O <header> é um conteiner. Dentro dele, você normalmente coloca os iten introdutórios, como:

  • Logo do site (uma <img> dentro de um link <a>).
  • Título principal da página (<h1>).
  • Menu de navegação (usando a tag <nav>).
  • Slogan do site (um <p> ou <span>).
  • Um formulário de busca.
<nav>

A função principal da tag <nav> (abreviação de "navigation" ou navegação) é identificar um bloco de links que serve como navegação principal do seu site ou de uma página.

O que vai dentro da tag <nav>?

Quase sempre o conteúdo de uma tag <nav> é uma lista não ordenada (<ul>) de links (<a>).

HTML 

<nav>
    <ul>
        <li><a href="caminho/home.html">Página Inicial</a>
        <li><a href="caminho/sobre.html">Sobre Nós</a>
        <li><a href="caminho/contato.html">Contato</a>
    </ul>
</nav>
Onde a tag <nav> é usada?

A tag <nav> é usada para navegação principal. Os locais mais comuns são:

  1. Dentro do <header>: este é o uso mais frequente. O <header> contém o logo e a navegação principal, e a <nav> envolve os links do menu.
  2. Como uma barra lateral: se o seu site tem uma navegação lateral, a tag <aside> pode conter uma tag <nav>.
  3. Para navegação na página: em uma página longa, você pode ter um <nav> com um índice (links para seções da própria página, como href="#secao1").
Importante!

A tag <nav> não deve ser usado para qualquer grupo de links. Ela é reservada para blocos de navegação principais.

  • Evite usá-la para uma lista simples de links sociais no rodapé. O <footer> por si só já é seemântico o suficiente.
  • Evite usá-la para links de paginação (Próximo, Anterior, 1, 2, 3...).
  • Evite usá-la para links dentro do conteúdo de um artigo.
<main>

A tag <main> serve para envolver o conteúdo principal e único daquele documento HTML específico.

Diferentemente de <div>, <section> ou <article>, que pode usar quantas vezes quiser, a tag <main> tem uma regra especial: só deve exister UM elemente <main> visivel por página.

O que vai dentro da tag <main>?
  • O conteúdo exclusivo daquela página.
  • Artigos, seções de texto, imagens do conteúdo, vídeos principais...
O que não vai dentro da tag <main>?
  • Conteúdo que se repete em todas as páginas (logos, menus de navegação principais, avisos de copyright, rodapés globais).
  • Barras de laterais (<aside>) que contêm links de navegação ou publicidade genérica.
Onde se encaixa na estrutura?
HTML 

<body>
    <header>...</header>

    <nav>...</nav>

    <main>
        <h1>Título do Artigo</h1>
        <p>Conteúdo exclusivo desta página...</p>

        <section>
            <h2>Subtítulo</h2>
            <p>Mais conteúdo...</p>
        </section>
    </main>

    <footer>...</footer>
</body>
<footer>

A tag <footer> define o rodapé de um documento ou de uma seção. Ela contém informações de fechamento sobre o conteúdo que ela envolve.

Os Dois Lugares Onde o <footer> Vive

Assim como o <header>, o <footer> é versátil e pode ser usado em dois contextos diferentes:

  1. O rodapé da página global: quando o <footer> é filho direto do <body>, ele representa o radapé do site inteiro. Geralmente é a última coisa que aparece na tela.
  2. O radapé de uma seção (local): você pode colocar um <footer> dentro de um <article> ou <section>. Nesse caso, ele contém informações específicas apenas sobre aquele pedaço de conteúdo (como autor do artigo ou a data de publicação).
O Que Geralmente Vai Dentro de um <footer>?
  • Direitos autorais (Copyright): ©2025 Sua Empresa.
  • Informações de contato: e-mail, endereço físico.
  • Links legais: política de privacidade, termos de uso.
  • Redes sociais: links para Instagram, LinkedIn, GitHub.
  • Mapa do site: uma lista de links importantes para ajudar na navegação.
  • A tag <address>: é muito comum colocar a tag <address> (que define informações de contato) dentro do <footer>.
HTML 

<footer>
    <div class="social-links">
        <a href="#">Facebook</a>
        <a href="#">Instagram</a>
    </div>

    <p>© 2025. Todos os direitos reservados. </p>

    <address>
        Contato: suporte@exemplo.com
    </address>
</footer>
<article>

A tag <article> deve ser usada para envolver um conteúdo que faz sentido por si só, mesmo se você o retirasse do seu site e colocasse em outro lugar (como um leitor de RSS ou compartilhasse no WhatsApp).

    Exemplos de uso:
  • Um post de blog completo.
  • Uma notícia.
  • Um comentário de um usuário em um fórum.
  • Um cartão de produto em uma loja virtual (tem foto, preço e nome.).
<section>

A tag <section> serve para agrupar conteúdos que compartilham um mesmo tema. Ela é uma divisão lógica do seu documento.

Diferente do <article>, uma <section> geralmente não faz sentido se estiver isolada. Ela faz parte de um todo.

    Exemplos de uso:
  • Capítulos de um texo.
<aside>

A tag <aside> é para conteúdo que está relacionado ao conteúdo principal, mas que não é o foco principal. Se você remover o <aside>, o conteúdo principal ainda deve fazer sentido.

Pense nele como aquelas caixas laterais em revistas com "Curiosidadees" ou "Saiba mais".

    Exemplos de uso:
  • Barras laterais (sidebars): menus laterais, listas de categorias, propaganda.
  • Box de informações: um quadro "Você sabia?" dentro de um artigo.
  • Glossário: explicação de um termo difícil ao lado do texto.
HTML 

<body>
    <header>...</header>

    <nav>...</nav>

    <main>
        <article>
        
            <h1>Título do Artigo</h1>
            <p>Conteúdo exclusivo desta página...</p>

            <section>
                <h2>Subtítulo</h2>
                <p>Mais conteúdo...</p>
            </section>

        </article>
    </main>

    <footer>...</footer>
</body>
Tags Não Semânticas
<div>

A <div> é um elemento coringa em bloco (block-level).

  • Comportamento: ela ocupa 100% da largura disponível e força um quebra de linha antes e depois dela. De forma geral, ela "empilha" elementos.
  • Analogia: imagine empilhar caixas de papelão. Você joga vários itens dentro da caixa. Quando você coloca uma caixa no chão, ela ocupa o espaço dela e a próxima caixa tem que ir embaixo ou em cima (a menos que você mude isso com CSS).
  • Uso principal: criar layouts, agrupar grandes seções, criar colunas, "embrulhar" (wrap) conteúdo para centralizar.
<span>

O <span> é um elemento de linha (inline-level).

  • Comportamento: ele ocupa apenas a largura do seu conteúdo e NÃO quebra linha. Ele flui junto com o texto.
  • Analogia: imagine uma caneta marce-texto. Você usa para destacar uma palavra ou frase dentro de um parágrafo, sem quebrar o texto ou criar um novo parágrafo.
  • Uso principal: estilizar pedaços específicos de texto (mudar cor, fonte) ou colocar ícones ao lado de palavras.
