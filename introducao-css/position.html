position

Serve para tirar elementos do fluxo normal da página e coloca-los exatamente onde você quiser.

position: static; (o padrão)

Todo elemento HTML nasce assim. Ele segue o fluxo normal da página (um bloco embaixo do outro).

  • Detalhe: top, left, z-index, etc., Não funcionam aqui. Se você tentar usar, nada acontece.

position: relative; (o ajuste fine)

O elemento continua ocupando seu espaço original no fluxo, mas você pode "empurrá-lo" visualmente para os lados, ou seja, ele se movimenta em relação a ele mesmo.

  • Comportamento: se você usar top:20px ele desce 20px em relação a onde ele deveria estar.
  • O fantasma: o lugar original dele fica vazio, preservado, como se ele ainda estivesse lá.
  • Uso principal: ele serve como âncora para filhos como position: absolute

position: absolute; (o rebelde)

Este elemento é removido completamente do fluxo. É como se ele passasse a flutuar por cima da página. Os outros elementos agem como se ele não existesse e ocupam o seu lugar.

  • Regra de ouro: o elemento absoluto vai se posicionar em relação ao primeiro pai que não seja static, ou seja, ele se movimenta em relação ao pai que tiver posicionado.
    • se o pai tiver position: relative, o filho absoluto fica preso dentro do pai.
    • se nenhum pai tiver posição definida, ele vai se prender lá no topo do body.

position: fixed; (o adesivo)

O elemento é removido do fluxo e posicionado em relação a janela do navegador, viewport.

  • Comportamento: ele fica colado na tela. Se você rolar a página, o elemento não se mexe.
  • Uso: usado para menus, dessa forma ele fica sempre visível o topo, não importa o quanto você desça a página.

position: sticky; (o híbrido)

É uma mistura de relative e fixed.

  • Comportamento: o elemento age normalmente (rola junto com a página) até atingir uma posição específica (ex: top:0) Quando ele chega lá, ele gruda e vira fixed.
  • Uso: muito usado em tabelas (para fixar o cabeçalho) ou em barras laterais de anúncios que rolam até certo ponto e depois param.

Dica de ouro: contexto de empilhamento do z-index (stacking context)

O z-index (que controla quem fica em cima de quem) só funciona se o elemento tiver position definido como relative, absolute ou fixed. Elementos static ignoram o z-index.

Muita gente acha que o z-index é uma competição global, onde quem tem o maior número ganha. Não é bem assim.

O z-index funciona dentro de hierarquias.

Imagine duas caixas (Pai A e Pai B).

  • Se o Pai A tem z-index:10;.
  • E o Pai B tem z-index:20;.

O Pai B vai cobrir o Pai A. E aqui está o segredo:

  • Se você tiver um elemento filho dentro do Pai A com z-index:9999;, ele AINDA VAI FICAR ATRÁS do Pai B.

Por que? Porque o filho está preso dentro do nível do pai. Se o pai perdeu a briga de altura, o filho perde junto, não importa quão alto seja o número dele. Ele é o rei... Mas apenas dentro do castelo dele.

Boas Práticas (evite a guerra dos 9)

Evite fazer isso:

CSS 

.header {
    z-index: 999999;
}

.modal {
    z-index: 9999999;
}

Isso vira bagunça impossível de manter. Tente criar uma escala lógica no seu projeto:

  • Camada base (conteúdo): 1.
  • Camada de decorações: -1.
  • Dropdowns e menus: 100.
  • Header fixo: 1000.
  • Modais/Popups: 2000.
